This page is protected with a custom servlet filter.

  • The upside is that the servlet filter code is independent of the application server.
  • The downside is that the application server is unaware of the logged user.

The implementation is comprised mainly of the following:

  • web.xml <filter> - declares the protection filter
  • web.xml <filter> <init-param> - declares the login form file
  • test.view.auth.custom.RequireLoginFilter - the protection filter that is called upon an un-authenticated request
  • test.core.auth.UserService(Bean) - the actual application specific authentication logic
